# Security Policy

## Supported Versions

Security support is provided for the latest repository state on the default branch.

## Reporting a Vulnerability

If you discover a security issue, please do not disclose it publicly in an issue.

Instead, report privately with:

- a clear description of the vulnerability,
- affected files or components,
- reproduction steps or proof of concept,
- potential impact assessment.

## Response Expectations

Maintainers will:

1. Acknowledge receipt of the report.
2. Assess severity and scope.
3. Determine remediation priority.
4. Publish a fix and disclosure note when appropriate.

## Scope Notes

This repository is primarily a methodology and analytics framework. Security-sensitive areas may include:

- automated workflows and CI configuration,
- external data ingestion logic (when added in future versions),
- dependency updates and third-party integrations.
