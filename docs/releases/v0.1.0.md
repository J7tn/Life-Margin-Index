# Release Notes: v0.1.0

Release date: 2026-03-16

## Overview

Version `0.1.0` establishes the first validation-ready release of the Life Margin Index repository. This release introduces the full conceptual framework, reproducible computation path, automated test coverage for core logic, and governance standards for methodology updates.

## Highlights

- Initial public framework and academic documentation for LMI.
- Executable metric functions with input validation in `src/`.
- Reproducible empirical demo pipeline in `analysis/`.
- Automated test suite for core metric and baseline calculator behavior.
- Citation metadata and bibliography for research usage.
- Methodology versioning and changelog governance.
- Community templates for issues and pull requests.
- Contributor guide and CI workflow for automated validation.
- Repository governance files for ownership, conduct, security, and review policy.
- Unified CLI now includes `validate`, `compute`, and `report` data pipeline commands.

## Validation Status

- Empirical demo run: success
- Unit tests: success (`8/8` passing)
- Linter diagnostics on edited Python files: no errors

## Known Limitations

- Empirical demonstration currently uses synthetic data only.
- Extension indices (Stability, Buffer, Debt Drag, Time Cost) are conceptually defined but not yet implemented in source modules.
- No packaged distribution artifact yet (`pyproject.toml` and published package pipeline pending).

## Recommended Next Milestones

1. Integrate pilot regional data with documented source provenance.
2. Add visualization-generating scripts (distribution, subgroup, and time-series charts).
3. Implement extension index modules and corresponding tests.
4. Add CI workflow to run tests automatically on pull requests.
